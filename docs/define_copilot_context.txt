📌 1. Project Overview

DMAIC Copilot adalah aplikasi Agentic AI berbasis Python + Streamlit yang berfungsi sebagai
digital Continuous Improvement engineer untuk menjalankan proyek Lean Six Sigma DMAIC
(Define, Measure, Analyze, Improve, Control).

Sistem ini dirancang untuk:
- Menjalankan logika DMAIC secara terstruktur
- Menggunakan LLM sebagai reasoning & synthesis engine
- Menyimpan state proyek secara persisten
- Mendukung iterative draft → feedback → final workflow
- Menyediakan laporan standar DMAIC tanpa regenerasi ulang yang tidak perlu

---------------------------------------------------------

📌 2. Core Design Principles (IMPORTANT)

1️⃣ Single Project State Authority  
Untuk setiap project dan setiap fase (DEFINE, MEASURE, dst), hanya ada SATU state resmi
yang dianggap valid oleh sistem.

2️⃣ Draft vs Final Separation (LOCKED DESIGN)
Untuk fase DEFINE:
- Draft disimpan terpisah dari Final
- Draft boleh berubah berkali-kali
- Final bersifat IMMUTABLE (tidak berubah kecuali user eksplisit melakukan revise + finalize ulang)

3️⃣ Agent ≠ UI  
- Agent (define_agent.py) bertanggung jawab atas reasoning, synthesis, dan penyimpanan state
- UI (1_DEFINE.py) hanya memicu agent dan menampilkan hasil
- UI tidak pernah menulis ke disk secara langsung

4️⃣ Download ≠ Finalize  
- Download report TIDAK memicu LLM
- Finalize adalah aksi eksplisit yang mengunci hasil

🎯 Primary Goals

📌 4. System Architecture Summary

🔹 A. Data Processing Layer
Melakukan analitik DMAIC (akan berkembang per fase):
- CTQ extraction
- VOC structuring
- Defect / performance summary
- Pareto & RCA support (Measure / Analyze)
- DOE planning (Improve)

🔹 B. AI Agent Layer

Setiap fase memiliki agent terpisah.

DEFINE PHASE (current focus):
- File: app/agents/define_agent.py
- Fungsi utama:
  - run_define_agent()
  - finalize_define_agent()

Agent DEFINE melakukan:
1. Perception
   - Interpretasi input user
   - Retrieval memory (GLOBAL_DEFINE)
   - Audit similarity
2. Decision
   - Penentuan CTQ, Y variable, scope, risk
3. Action
   - Penyusunan DEFINE package (problem, goal, SIPOC, dsb)
4. Self-critique & Adaptation (saat finalize)
5. Insight generation (LLM-based)

🔹 C. UI Layer (Streamlit)

DEFINE UI:
- Create new project
- Open existing project (FINAL only)
- Generate Draft
- Feedback loop (Generate Draft ulang)
- Finalize Draft
- Download Final Report
- Audit Log Viewer

📌 5. Draft–Final Workflow (DEFINE PHASE)

### A. New Project Flow
1. User memasukkan input DEFINE
2. User klik "Generate Draft"
3. Agent menghasilkan DEFINE Draft
4. Draft disimpan ke:
   <project_id>_define_draft.json
5. User:
   - Setuju → klik Finalize
   - Tidak setuju → beri feedback → Generate Draft ulang

### B. Finalization Flow
1. User klik "Finalize Draft"
2. Agent menjalankan self-critique & adaptation
3. Final DEFINE dikunci
4. Final disimpan ke:
   <project_id>_define_final.json
5. Draft dihapus

### C. Load Existing Project
- Hanya FINAL yang di-load
- Tidak ada LLM call
- Tidak ada perubahan state
- Report bisa di-download kapan pun tanpa finalize ulang

### D. Revise Existing Final
- User klik "Revise"
- Final dipakai sebagai baseline
- Draft baru dibuat lewat Generate Draft
- Final lama tetap utuh sampai finalize ulang

📌 6. Persistent Memory Design

📁 Storage Location:
dmaic_memory/

DEFINE files:
- <project_id>_define_draft.json   → draft aktif (opsional)
- <project_id>_define_final.json   → FINAL resmi (authoritative)

GLOBAL memory:
- define_global.json
  - Berisi ringkasan episode DEFINE
  - Digunakan untuk few-shot & similarity retrieval
  - BUKAN sumber UI state

📌 11. Glossary

Agent  
= Python module executing one DMAIC phase using LLM reasoning

Draft  
= Mutable intermediate result, subject to feedback

Final  
= Locked, authoritative result for a phase

Project Memory  
= JSON files storing draft/final state per phase

GLOBAL_DEFINE  
= Cross-project memory for retrieval & learning

Audit Log  
= Chronological trace of agent actions per phase

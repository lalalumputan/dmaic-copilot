📌 4. Agent Architecture Pattern (all phases)
=========================================================

run_*_agent():
  1. PERCEPTION  — read upstream context + similar episodes from memory
  2. DECISION    — LLM decides strategy (JSON output)
  3. ACTION      — LLM generates full phase package (JSON output)
  4. GATE        — deterministic rules-based evaluation (no LLM)
  5. COACHING    — deterministic coaching markdown (no LLM)
  → save draft to SQLite

finalize_*_agent():
  - NO LLM calls
  - Gate must not be FAILED
  - Saves final to SQLite, deletes draft
  - Resets approvals if re-finalizing after rejection

Governance flow:
  project_leader → generate draft → submit (finalize)
  reviewer       → approve/reject
  champion       → approve/reject
  gate OPEN      → project_leader can advance to next phase

=========================================================
📌 5. Key Rules for Coding Sessions
=========================================================

1. Logic in agents, NOT in pages — pages are UI only
2. LLM called ONLY in run_*_agent() — never in finalize or UI callbacks
3. All persistence via memory.py SQLite functions — never write files directly
4. Single source of truth: define_state["outputs"] for all report rendering
5. Patch minimal — do not rewrite entire files unless explicitly requested
6. Session state keys are project-specific where needed: f"key_{active_pid}"
7. st.data_editor must be OUTSIDE st.form() to avoid form context errors
8. Always check _loaded_pid before auto-loading from disk
9. VOC/VOB table key: f"define_voc_table_{active_pid}"
10. secrets.toml, *.docx, dmaic_memory.db must stay in .gitignore

📌 8. Glossary
=========================================================

Agent        = PAD-Gate module for one DMAIC phase
draft        = unfinalized phase state (editable)
final        = locked phase state (immutable until rejected)
gate         = deterministic rules check before finalize
approval     = reviewer/champion sign-off after finalize
episode      = saved final state for few-shot memory retrieval
coaching_md  = deterministic guidance markdown (no LLM)
insight_md   = same as coaching_md (legacy alias)
summary_md   = full phase report in markdown (stored in state)
active_pid   = st.session_state["active_project_id"]
_loaded_pid  = st.session_state["_loaded_pid"] (tracks last loaded project)

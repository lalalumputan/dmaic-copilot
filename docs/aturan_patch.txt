

### 1) Tidak ada lagi “replace seluruh blok” kecuali 1 file full-replace

* Default: **PATCH MINIMAL** → hanya *insert* di titik yang jelas.
* Full replace hanya boleh kalau kamu **minta eksplisit**: “full replace file X”.

### 2) Semua patch harus punya 3 bagian (format tetap)

1. **Target file**: mis. `app/pages/1_DEFINE.py`
2. **Cari anchor unik**: 1–2 baris yang pasti ada di kodenya (contoh: `if st.sidebar.button("📂 Load Existing DEFINE"`).
3. **Aksi**: `INSERT BELOW`, `INSERT ABOVE`, atau `REPLACE THIS EXACT BLOCK (from..to..)`.

> Dengan ini kamu tidak perlu menebak “tempel di mana”, dan tidak akan double.

### 3) Tidak boleh bikin nama variabel baru kalau sudah ada padanan

* Kalau kamu bilang `voc_raw` sudah ada, maka patch harus **pakai `voc_raw`**, bukan `voc_list`.
* Kalau nama tidak pasti, patch harus pakai `st.session_state[...]` langsung.

### 4) Satu sumber kebenaran (Single Source of Truth)

* Form values: **selalu** lewat `st.session_state[key]` (bukan `value=` di widget).
* Output report: **selalu** dari `define_state.summary_md` (atau `reporting.build_define_summary_md(state)`).
* Memory load: selalu mengisi `st.session_state` via satu fungsi prefill.

### 5) Satu tombol = satu efek

* Generate/Update Draft → hanya `run_define_agent()`
* Finalize → hanya `finalize_define_agent()`
* Generate Word → hanya `reporting/export` (tanpa Llm regeneration)

### 6) Setiap perubahan harus menyertakan “rollback”

* “Jika error, undo hanya di 3 baris ini.” (aku akan tulis baris mana yang diubah)

---

## Agar aturan ini jalan, kita butuh 1 kebiasaan kecil dari kamu (paling hemat effort)

Setiap minta perubahan, kirim **1 cuplikan kecil (20–40 baris)** yang memuat:

* bagian tempat patch akan masuk (anchor)
* bukan seluruh file

Dengan begitu aku bisa kasih patch yang **100% cocok** dengan kode yang benar-benar ada, tanpa asumsi.
